## 2. Reflection: Building Websites with AI

Building a website with AI was a helpful and interesting experience. I used AI as a guide for planning the structure, improving the design, and checking whether the page met the project requirements. This was one of my first times using AI to support a web development project, so I was surprised by how quickly it could turn an idea into working HTML and CSS.

At the same time, I learned that AI does not replace my own understanding. I still needed to read the code, make design decisions, edit the content, and test whether the navigation and layout worked correctly. The process helped me understand the difference between HTML and CSS more clearly. HTML gives the page its structure and meaning, while CSS controls the visual style, spacing, layout, and overall mood of the site.

I also learned that a simple design can still feel intentional. For this project, I chose a black-and-white minimalist style instead of using many colors. That forced me to focus on typography, spacing, borders, and layout. Overall, AI made the project less intimidating, but I still had to make the website personal and functional.

## 3. Interview Question Plan: “Can you explain how HTML and CSS work together?”

I would answer this question by explaining that HTML and CSS have different but connected roles.

First, HTML is the structure of a webpage. It defines the content and sections, such as headings, paragraphs, navigation links, images, project cards, and contact information. HTML tells the browser what each part of the page means.

Second, CSS is the styling layer. It controls how the HTML looks on the screen, including colors, fonts, spacing, borders, alignment, and responsive layout. CSS can select HTML elements by tag name, class, or ID, and then apply visual rules to them.

For example, in my portfolio website, the HTML creates sections like hero, about, skills, projects, and contact. The CSS turns those sections into a black-and-white minimalist design by adding a grid background, large typography, borders, spacing, and responsive layouts. So HTML is like the skeleton or content of the page, and CSS is like the design system that makes it visually organized and engaging.
